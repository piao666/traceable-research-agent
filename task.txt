# Traceable Research Agent - Task Plan

更新时间：2026-06-29

## 项目定位

Traceable Research Agent 的核心目标是构建一个可追踪、可审计、可复盘的多工具调研 Agent。当前主线不是扩展第二套前端，也不是转向企业知识库 RAG 平台，而是继续强化：

- 任务规划与工具调用的稳定性
- Trace / Evidence / Report 的闭环
- MCP 只读工具协议互操作
- 真实演示问题下的端到端可靠性

Streamlit 继续作为主 demo UI；FastAPI 是稳定后端；MCP 端点用于外部客户端发现和只读调用。

## 当前已具备的闭环

- FastAPI 任务生命周期：create / plan / run / run_async / confirm / status
- Planned 与 ReAct 两种执行模式
- Tool Registry 统一注册本地工具和可选远端 MCP 工具
- file_reader / sql_query / rag_search / mcp_github_search / tavily_search / report_writer
- SQLGlot 只读校验、文件路径白名单、GitHub GET-only、HITL、ReAct 循环上限
- SQLite AgentRun / ToolTrace 持久化
- SSE 实时 Trace 事件流
- Streamlit 任务创建、计划查看、执行追踪、报告渲染
- EvidenceBundle 聚合、多来源证据分组、mock/fallback/failure 标记
- Evidence Export：JSON / JSONL / Markdown metadata、content preview、download
- MCP Server：只读工具发现、工具调用、trace_reader、report_reader
- MCP Client / Remote Tool Registry：可选发现安全远端只读工具

## 2026-06-29 - Day43.5 Stabilization

背景：深度排查 Day40-Day43 后发现核心 API、Streamlit、SSE、MCP、Evidence Export 都能闭环运行，但真实 LLM Planner 在自由问题下可能生成不匹配本地工具边界的参数，例如：

- `file_reader.path` 带 `workspace/docs/` 前缀或引用不存在文件
- `sql_query.query` 查询 demo schema 中不存在的列或表
- `mcp_github_search.query` 过长、包含混乱换行或无效参数，可能触发 GitHub Search HTTP 422

修复目标：

- 在计划落库前对工具参数做语义归一化
- 保持工具自身安全边界不放松
- 不新增重型依赖
- 不破坏 Planned、parallel、ReAct、Trace、SSE、MCP、Evidence Export

实现内容：

- 新增 `app/agent/plan_guardrails.py`
  - `normalize_plan_arguments(plan, task, source_mode)`
  - 文件路径归一化到 `workspace/docs` 下已存在相对路径
  - 文件缺失时按任务关键词选择已有 demo 文档，默认回退 `demo_research_note.md`
  - SQL 只读校验后用 SQLite `EXPLAIN QUERY PLAN` 预检 schema
  - SQL 无效时按任务语义回退到 demo schema 下的安全 SELECT
  - GitHub query 压缩为短 plain text
  - GitHub repo/search_type/sort/order/limit/mode 归一化
  - RAG/Tavily 缺省 query、top_k/max_results、mode 参数归一化
  - report_writer 参数清空
  - 归一化原因写入 `plan["notes"]`
- 修改 `app/agent/planner.py`
  - deterministic plan 和 LLM plan 返回前统一调用 guardrail
  - LLM 子问题分解后再次归一化，防止后处理引入参数偏移
- 修改 `app/llm/planner_client.py`
  - Planner prompt 明确列出 workspace/docs 文件边界
  - 明确 demo SQLite schema
  - 明确 GitHub query/repo/search_type/read-only 边界
- 新增 `scripts/smoke_planner_guardrails.py`
  - 模拟坏 LLM plan
  - 覆盖文件前缀修正、SQL schema 修正、GitHub 参数修正、RAG 参数修正、report_writer 参数清理
  - 真实调用 file_reader、sql_query、GitHub mock 工具确认修正后可执行
- 修改 `scripts/smoke_final_project.py`
  - 聚合验证加入 `smoke_planner_guardrails.py`
- 修改 `README.md`
  - 重写为正常项目 README
  - 删除阶段流水账、过时限制和旧 MCP 定位表述
- 修改 `task.txt`
  - 删除早期历史规划和旧前端扩展路线
